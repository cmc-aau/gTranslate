fit_models

fit_models takes the features built from build_features and trains the model to recognize the boundaries between Table 11, 4, and 25. Once it’s done, you’ll have fresh model files ready for the detect_table command.

Arguments

usage: gtranslate fit_models --feature_file FEATURE_FILE --tt_file TT_FILE
                             --out_dir OUT_DIR [--cpus CPUS] [--seed SEED]
                             [--split_data] [-h]

required named arguments

--feature_file

path to TSV file containing features for each genome

--tt_file

File indicating the translation table for each genome.

--out_dir

directory to output files

optional arguments

--cpus

number of CPUs to use

Default: 1

--seed

seed for reproducibility

--split_data

Enable data splitting into training and validation sets.
